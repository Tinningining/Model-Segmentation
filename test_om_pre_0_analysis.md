## 11. 数据含义解读

### 为什么只有 Layer 0 Head 0 有数据？

OM decode 模型的 `present_key` 输出是 `cat(past_key_input, new_key)`，shape 为 `(7, 1, 8, 1025, 128)`。其中：

- Position 0 ~ past_len-1：来自输入的 `past_key`（KV cache）
- Position past_len：新 token 计算出的 key
- Position past_len+1 ~ 1024：未使用的空位（零）

从数据看，past_len = 55（position 0-54 是 past KV，position 55 是新 token），但只有 Layer 0 Head 0 有数据。这说明 **输入给 OM 模型的 past_key 本身就几乎全是零**。

### 根本原因

在 prefill 阶段，代码从 OM 模型输出中提取 present_key 时使用了错误的切片方式：

```python
# 错误代码（原始）
present_key = present_key_full[:, :, :, -q_len:, :]
# 取的是 position (512-q_len) ~ 511，即 padding 区域（全零）

# 正确代码（修复后）
present_key = present_key_full[:, :, :, :q_len, :]
# 取的是 position 0 ~ q_len-1，即有效数据区域
```

Prefill 时 OM 输出 shape 为 `(7, 1, 8, 512, 128)`，有效数据在前 q_len 个位置（如 position 0-54），但 `-q_len:` 取的是末尾的 q_len 个位置（如 position 457-511），这些位置是 padding 产生的零值。

因此写入 KV cache 的数据几乎全是零，导致 decode 阶段 OM 模型收到的 past_key 输入也几乎全是零，最终输出的 present_key 也只有极少量有效数据。
