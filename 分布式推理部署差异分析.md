# Qwen 与 Llama 分布式推理部署差异深度分析

## 概述

本文档深入分析 Qwen 和 Llama 两个模型在分布式推理部署架构上的核心差异。Qwen 采用专门设计的分布式推理框架（`distributed_inference`），而 Llama 则通过会话抽象（`inference_net`）实现分布式能力。

---

## 1. 架构设计理念

### 1.1 Qwen 的架构理念

**核心思想：专用分布式框架，节点角色明确**

```
设计原则：
┌─────────────────────────────────────────────────────────────┐
│  1. 节点角色专业化 - 每个节点有明确的职责                      │
│  2. 消息驱动架构 - 使用结构化消息进行通信                      │
│  3. 流水线并行 - 真正的流水线式数据流                         │
│  4. 独立 KV Cache - 每个节点管理自己的 KV Cache               │
└─────────────────────────────────────────────────────────────┘
```

**节点类型：**
- **HeadNode (node_head.py)**: 头节点，负责 embedding + layers_0_6
- **MiddleNode (node_middle.py)**: 中间节点，负责 layers_7_13 或 layers_14_20
- **TailNode (node_tail.py)**: 尾节点，负责 layers_21_27 + output

### 1.2 Llama 的架构理念

**核心思想：会话抽象，统一接口**

```
设计原则：
┌─────────────────────────────────────────────────────────────┐
│  1. 会话抽象 - 通过 Session 类统一不同推理模式                │
│  2. 模式切换 - 单机/分布式通过配置切换                        │
│  3. 兼容性优先 - 保持与单机推理相同的接口                     │
│  4. 渐进式改造 - 在现有架构上添加分布式能力                   │
└─────────────────────────────────────────────────────────────┘
```

**会话类型：**
- **OnnxSession**: ONNX Runtime 推理
- **AclSession**: 昇腾 ACL 推理（单机或切分模型）
- **NetSession**: 网络分布式推理

## 2. 节点通信机制

### 2.1 Qwen 的通信机制

**特点：结构化消息 + 双向通信**

#### 消息封装

```python
# qwen/distributed_inference/network.py

class DistributedMessage:
    """分布式消息封装"""
    
    # 消息类型
    MSG_FORWARD = "forward"      # 前向传播数据
    MSG_RESULT = "result"        # 结果返回
    MSG_CONTROL = "control"      # 控制消息
    MSG_RESET = "reset"          # 重置 KV Cache
    MSG_SHUTDOWN = "shutdown"    # 关闭节点
    
    def __init__(self, msg_type: str, step: int, data: dict):
        self.msg_type = msg_type
        self.step = step
        self.data = data  # 包含 hidden_states, attention_mask, position_ids, meta
```

#### 通信流程

```
前向传播流程：
┌──────────┐  MSG_FORWARD   ┌──────────┐  MSG_FORWARD   ┌──────────┐
│  Node 0  │───────────────▶│  Node 1  │───────────────▶│  Node 2  │
│  (Head)  │                │ (Middle) │                │ (Middle) │
└────┬─────┘                └──────────┘                └──────────┘
     │                                                         │
     │                                                         ▼
     │                                                   ┌──────────┐
     │                                                   │  Node 3  │
     │                                                   │  (Tail)  │
     │                                                   └────┬─────┘
     │                                                        │
     │                      MSG_RESULT                        │
     └────────────────────────◀──────────────────────────────┘
                         (next_token)

数据传输内容：
- Forward: hidden_states [1, 16, 2048] + attention_mask + position_ids + meta
- Result: next_token (单个整数)
```

#### 网络组件

```python
# 每个节点都有：
class Node:
    def __init__(self):
        self.server = NodeServer(port)      # 接收上游数据
        self.client = NodeClient(ip, port)  # 发送下游数据
```

**优势：**
- ✅ 消息类型明确，易于调试
- ✅ 支持控制消息（reset, shutdown）
- ✅ 元数据传递（past_len, q_len 等）
- ✅ 双向通信，结果直接返回头节点

### 2.2 Llama 的通信机制

**特点：简单序列化 + 单向传递**

#### 消息传输

```python
# llama/inference_net/session.py

def send_msg(sock, msg):
    """发送消息列表（numpy 数组）"""
    data = pickle.dumps(msg)
    sock.sendall(struct.pack('>I', len(data)))
    sock.sendall(data)

def recv_msg(sock):
    """接收消息"""
    raw_msglen = recvall(sock, 4)
    msglen = struct.unpack('>I', raw_msglen)[0]
    data = recvall(sock, msglen)
    return pickle.loads(data)
```

#### 通信流程

```
NetSession 流程（4 节点）：
┌──────────┐                ┌──────────┐                ┌──────────┐
│  Node 1  │───────────────▶│  Node 2  │───────────────▶│  Node 3  │
│  (M0)    │  [hidden, ...]  │  (M1)    │  [hidden, ...]  │  (M2)    │
└────┬─────┘                └──────────┘                └──────────┘
     │                                                         │
     │                                                         ▼
     │                                                   ┌──────────┐
     │                                                   │  Node 4  │
     │                                                   │  (M3)    │
     │                                                   └────┬─────┘
     │                                                        │
     │                      [logits]                          │
     └────────────────────────◀──────────────────────────────┘

数据传输内容：
- Forward: [hidden_states, input_ids, mask, pos_ids, past_key_values_0]
- Result: logits 或 [logits]
```

#### 网络设置

```python
class NetSession(Session):
    def __init__(self, config):
        # 1. 启动服务器（接收返回结果）
        self.server_sock = socket.socket(...)
        self.server_sock.bind(('0.0.0.0', self.listen_port))
        self.server_sock.listen(1)
        
        # 2. 连接到下一个节点
        self.client_sock = socket.socket(...)
        self.client_sock.connect((self.next_ip, self.next_port))
        
        # 3. 接受来自最后节点的连接
        self.prev_conn, addr = self.server_sock.accept()
```

**劣势：**
- ❌ 无消息类型区分
- ❌ 无控制消息机制
- ❌ 需要手动管理连接顺序
- ❌ 调试困难（数据格式不明确）

## 3. 节点实现对比

### 3.1 Qwen 头节点实现

```python
# qwen/distributed_inference/node_head.py

class HeadNode:
    """头节点：embed + layers_0_6"""
    
    def generate(self, prompt_ids, max_new_tokens):
        """生成文本"""
        for i in range(max_new_tokens):
            # 1. 运行 embedding
            hidden = self.run_embed(current_ids)
            
            # 2. 构建 attention mask 和 position ids
            attention_mask = build_attention_mask(...)
            position_ids = build_position_ids(...)
            
            # 3. 运行 block (layers 0-6)
            hidden = self.run_block(hidden, attention_mask, position_ids, q_len)
            
            # 4. 构建元数据
            meta = {
                "past_len": self.past_len,
                "q_len": q_len,
                "max_cache_len": self.config.max_cache_len,
                "max_input_len": self.config.max_input_len,
            }
            
            # 5. 发送到下一个节点
            msg = DistributedMessage.create_forward_msg(
                step=self.step,
                hidden=hidden,
                attention_mask=attention_mask,
                position_ids=position_ids,
                meta=meta
            )
            self.client.send(msg.to_dict())
            
            # 6. 等待尾节点返回 token
            result = self.server.recv()
            result_msg = DistributedMessage.from_dict(result)
            next_token = result_msg.data.get("next_token")
            
            # 7. 更新状态
            self.past_len += q_len
            current_ids = np.array([[next_token]])
```

**特点：**
- ✅ 完整的生成循环控制
- ✅ 元数据管理
- ✅ 状态跟踪（past_len, step）
- ✅ 结构化消息处理

### 3.2 Qwen 中间节点实现

```python
# qwen/distributed_inference/node_middle.py

class MiddleNode:
    """中间节点：layers_7_13 或 layers_14_20"""
    
    def process_loop(self):
        """处理循环"""
        while True:
            # 接收消息
            data = self.server.recv()
            msg = DistributedMessage.from_dict(data)
            
            if msg.msg_type == DistributedMessage.MSG_FORWARD:
                # 提取数据
                hidden = msg.data["hidden"]
                attention_mask = msg.data["attention_mask"]
                position_ids = msg.data["position_ids"]
                meta = msg.data.get("meta", {})
                
                # 运行 block
                hidden_out = self.run_block(hidden, attention_mask, position_ids, q_len)
                
                # 转发到下一个节点
                forward_msg = DistributedMessage.create_forward_msg(...)
                self.client.send(forward_msg.to_dict())
                
            elif msg.msg_type == DistributedMessage.MSG_RESET:
                self.reset()
                self.client.send(msg.to_dict())
                
            elif msg.msg_type == DistributedMessage.MSG_SHUTDOWN:
                self.client.send(msg.to_dict())
                break
```

**特点：**
- ✅ 消息驱动架构
- ✅ 支持多种消息类型
- ✅ 自动转发机制
- ✅ 独立的处理循环

### 3.3 Llama NetSession 实现

```python
# llama/inference_net/session.py

class NetSession(Session):
    """网络分布式会话（仅在 Node 1 运行）"""
    
    def run(self, input_ids):
        """运行推理"""
        seq_len = input_ids.shape[-1]
        l, r = 0, self.max_len
        
        while l < seq_len:
            r = min(seq_len, r)
            current_batch_len = r - l
            
            # 1. 准备输入
            self.input_ids[:, :current_batch_len] = input_ids[:, l:r]
            cache, mask, pos_ids = self.kvCache.getInputs(self.max_len)
            
            # 2. 运行 M0 (embed + layers 0-4)
            acl_inputs = [self.input_ids[:, :current_batch_len], mask, pos_ids]
            acl_inputs.extend(cache)
            m0_res = self.model.inference(acl_inputs)
            
            hidden_states = m0_res[0]
            new_kvs = m0_res[1:]
            
            # 3. 更新本地 KV Cache
            self.kvCache.update(current_batch_len, new_kvs, None)
            
            # 4. 发送到 Node 2
            msg = [hidden_states, self.input_ids[:, :current_batch_len], 
                   mask, pos_ids, cache[0]]
            send_msg(self.client_sock, msg)
            
            # 5. 等待 Node 4 返回结果
            logits = recv_msg(self.prev_conn)
            
            l += self.max_len
            r += self.max_len
        
        return [logits]
```

**特点：**
- ❌ 仅在主节点运行
- ❌ 其他节点需要独立脚本
- ❌ 无统一的节点抽象
- ❌ 通信协议不明确

## 4. KV Cache 管理

### 4.1 Qwen 的 KV Cache 管理

**特点：每个节点独立管理**

```python
# 每个节点初始化自己的 KV Cache
self.kv_cache = create_kvcache(
    cache_type="basic",
    num_layers=self.config.get_num_layers(),  # 每个节点的层数
    num_kv_heads=self.config.num_key_value_heads,
    head_dim=self.config.head_dim,
    max_cache_len=self.config.max_cache_len
)

# Node 0: 7 层 (layers 0-6)
# Node 1: 7 层 (layers 7-13)
# Node 2: 7 层 (layers 14-20)
# Node 3: 7 层 (layers 21-27)
```

**KV Cache 格式：**
```python
# 输入/输出格式
past_key: [num_layers, batch, num_kv_heads, seq_len, head_dim]
past_value: [num_layers, batch, num_kv_heads, seq_len, head_dim]

# 例如 Node 0 (7层)
past_key.shape = [7, 1, 8, 1024, 128]
past_value.shape = [7, 1, 8, 1024, 128]
```

**更新机制：**
```python
def run_block(self, hidden, attention_mask, position_ids, q_len):
    # 获取 KV Cache
    past_key, past_value = self.kv_cache.get_cache()
    
    # 执行推理
    outputs = self.block_model.execute([
        hidden, attention_mask, position_ids,
        past_key, past_value
    ])
    
    # 更新 KV Cache
    present_key = outputs[1]
    present_value = outputs[2]
    self.kv_cache.update(present_key, present_value, q_len)
```

**优势：**
- ✅ 每个节点只管理自己的层
- ✅ 内存占用分散
- ✅ 无需网络传输 KV Cache
- ✅ 更新逻辑清晰

### 4.2 Llama 的 KV Cache 管理

**特点：主节点管理 + 传递依赖**

```python
# NetSession 只管理 M0 的 KV Cache (layers 0-4)
config.n_layer = 5  # 强制设置为 5 层
```

**传递机制：**

- ❌ 依赖关系复杂（RoPE 依赖第 0 层）

## 6. 错误处理和容错

### 6.1 Qwen 的错误处理

```python
# 连接重试
class NodeClient:
    def connect(self, retry_interval=1.0, max_retries=60):
        for i in range(max_retries):
            try:
                self.sock.connect((self.host, self.port))
                return True
            except ConnectionRefusedError:
                if i % 10 == 0:
                    print(f"Waiting for {self.host}:{self.port}...")
                time.sleep(retry_interval)
        return False

# 消息接收错误处理
def recv_msg(sock):
    try:
        raw_msglen = _recvall(sock, 4)
        if not raw_msglen:
            return None
        msglen = struct.unpack('>I', raw_msglen)[0]
        data = _recvall(sock, msglen)
        return pickle.loads(data)
    except Exception as e:
        print(f"[Network] Recv error: {e}")
        return None

# 节点关闭处理
def process_loop(self):
    while True:
        data = self.server.recv()
        if data is None:
            print(f"Connection closed")
            break
        
        msg = DistributedMessage.from_dict(data)
        
        if msg.msg_type == DistributedMessage.MSG_SHUTDOWN:
            self.client.send(msg.to_dict())
            break
```

**特点：**
- ✅ 自动重连机制
- ✅ 优雅关闭
- ✅ 错误日志
- ✅ 超时处理
